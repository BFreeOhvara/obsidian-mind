# CC Prompt Format

> When Eagle writes a prompt for Brayden to paste into Claude Code (CC), the prompt is the artifact — the chat is the commentary. Never mix them.

---

## The Rule

1. **Always an artifact, never inline.** Whenever Eagle (the Claude chat instance) writes a prompt that Brayden is meant to copy and paste into CC, Eagle must deliver it as a single clean artifact/file — never buried inside a chat reply between dashes or other text.

2. **The artifact contains ONLY the prompt.** No commentary, no explanations, no "here's what this does" — all of that stays in the chat message outside the artifact. What's in the artifact is exactly what gets pasted into CC, nothing more.

3. **One artifact per CC prompt.** If a session produces multiple CC prompts, each gets its own separate artifact — Brayden never has to figure out where one prompt ends and the next begins.

4. **Name each artifact descriptively** (e.g. `cc-context-alarm-prompt`, `cc-skills-reorg-prompt`) so they're identifiable later.

---

## CC Auto-Log Rule (permanent — no exceptions)

After EVERY completed task, CC automatically appends a log entry to `~/obsidian-mind/brain/Memories.md` formatted exactly:

```
[CC | YYYY-MM-DD HH:MM] — {what was done}
```

…and commits + pushes Atlas immediately. No exceptions, no batching across tasks, no "I'll log at session end." This keeps Atlas always current so [[reload]] always has fresh data — a /reload chain is only as good as the last log entry.

A task is not complete until its log entry is pushed.

---

## How the Rule Travels

Eagle only knows what's in its context block. Every OHVARA CONTEXT LOAD block generated by CC (per [[auto-handoff]] Thing 2) must include a one-line statement of this rule so every fresh Eagle chat enforces it from message one.
